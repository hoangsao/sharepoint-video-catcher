Sharepoint Video Catcher - Test Guide

This document will help you test the Sharepoint Video Catcher extension to ensure it's working correctly.

Test Environment Setup

Step 1: Load the Extension

Make sure the extension is loaded in your browser:

  • Open Chrome or Edge browser
  • Go to the extensions page (chrome://extensions/ or edge://extensions/)
  • Enable "Developer mode"
  • Click "Load unpacked" and select the extension folder
  • Verify that the Sharepoint Video Catcher extension appears in your extensions list

Step 2: Visit a Sharepoint Site

Visit a Sharepoint site that contains videos:

  • Log in to your organization's Sharepoint
  • Navigate to a page that contains video content
  • Play a video to trigger the network requests
Note: For testing, you can use any Sharepoint site with embedded videos. The extension should detect video manifest URLs that contain the string "videomanifest".

Testing the Extension

Step 3: Check for Video Detection

After playing a video on a Sharepoint page:

  • Click on the Sharepoint Video Catcher extension icon in the browser toolbar
  • The popup should display the detected video with its title
  • If no videos are detected, check the browser console for any error messages

Step 4: Test the Copy Functions

In the extension popup:

  • Click the "Copy URL" button and verify that the URL is copied to your clipboard
  • Click the "Copy Command" button and verify that the ffmpeg command is copied to your clipboard
  • Both buttons should show "Copied!" text when clicked

Step 5: Verify the ffmpeg Command

Test the generated ffmpeg command:

  • If you have ffmpeg installed, paste the copied command into a terminal/command prompt
  • Verify that the download starts and completes successfully
  • Check that the video file has been saved with the correct name
Important: The ffmpeg command format should be: ffmpeg -i "URL_WITHOUT_ENABLECDN" -codec copy "VIDEO_TITLE.mp4"

Troubleshooting

If the extension doesn't detect videos:

  • Make sure you're on a Sharepoint site (URL contains sharepoint.com)
  • Check that the video is actually playing (not just displayed as a thumbnail)
  • Open the browser's Developer Tools (F12) and check the Console for any error messages
  • In the Network tab, filter for "videomanifest" to verify if such requests are being made
